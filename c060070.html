6  Popular datasets

6.1 Getting started

You’ll often discover datasets through tutorials, course materials, or recommendations from others. If those sources aren’t enough, you may also search for datasets online or use your favorite large language model to suggest options.

When choosing a dataset, it’s useful to first consider where it comes from. Some datasets are included in the base R installation, while others are provided by add-on packages that must be installed separately. In particular, datasets in the datasets package are a good place to start, since this package is installed with R and is immediately available in every R session.

To help you get started, this chapter introduces several built-in datasets from the datasets package, along with a few examples from commonly used add-on packages. These datasets are well suited for learning and practicing R programming concepts.

6.2 datasets package

While official usage statistics for the built-in datasets are not published. The datasets shown below are generally considered as popular based on their frequent appearance in R tutorials, examples, and statistical literature.

6.2.1 iris

Description: This classic dataset, introduced by Sir Ronald Fisher, contains measurements in centimeters of the sepal length, sepal width, petal length, and petal width for 50 flowers from each of three species of iris: Iris setosa, versicolor, and virginica.

Popularity: It is arguably the most famous dataset in machine learning and statistics, widely used for introducing classification problems and concepts like cluster analysis.

6.2.2 mtcars

Description: This dataset comprises fuel consumption and 10 other aspects of automobile design and performance for 32 automobiles (1973–74 models) from the 1974 Motor Trend US magazine.

Popularity: It’s frequently used for illustrating regression analysis (especially multiple linear regression) and comparisons between different car characteristics.

6.2.3 USArrests

Description: This data set contains statistics about violent crime rates (Murder, Assault, and Rape) in each of the 50 US states in 1973, along with the percentage of the urban population.

Popularity: It’s a common example for demonstrating principal component analysis (PCA), cluster analysis, and other multivariate statistical techniques.

6.2.4 airquality

Description: This dataset provides daily air quality measurements in New York City from May to September 1973, including Ozone, Solar Radiation, Wind, and Temperature.

Popularity: It’s often used for learning about handling missing data (NA values), basic data exploration, and simple time-series analysis.

6.2.5 AirPassengers

Description: A classic time-series dataset recording the monthly total number of international airline passengers from 1949 to 1960.

Popularity: It’s an essential dataset for introducing time series analysis, including concepts like trend, seasonality, and forecasting models like ARIMA.

6.3 Recommended packages

Some packages that come with the standard R installation are not automatically loaded when an R session starts up. These packages are referred to as “recommended packages”. They are high-quality, widely-used packages the R Core Team consideres essential for most users. The MASS package is a famous example of this type of package. Others include lattice, mgcv and nlme. Below are details about MASS.

6.3.1 MASS

Here, I want to point out the MASS package because there’s sometimes confusion about this package for a couple of reasons.

Sometimes, people will refer to the MASS dataset, but it’s a package that contains datasets. This is similar to how the datasets package contains many datasets.

Be aware though, the MASS package contains many modern versions of functions that were previously used from the base R stats package. Think of stats as the standard toolbox and MASS as the professional upgrade—designed to handle messy, real-world data that violates the strict assumptions of classical statistics.

The MASS package is a recommended package, meaning it’s installed with base R, but you’ll need to run library(MASS) to make it available to your R session.

Below a few of the most frequently used MASS datasets:

  • Cars93: Data on 93 car models from 1993.

  • mammals: Brain and body weights for 62 species of land mammals.

  • birthwt: Risk factors associated with low infant birth weight.

  • crabs: Morphological measurements of Leptograpsus crabs.

Note

The Boston dataset was historically the most famous MASS dataset, but it was removed due to ethical concerns regarding the use of variables involving racial proportions. If needed, it can be found in the ISLR2 and spdep packages.

I’ve included this callout to draw attention that it’s sometimes appropriate to consider the sournce and content of data in built-in datasets.

6.4 Add-on packages

Packages that are not categorized as “base” or “recommened” are referred to as add-on packages. These packages are not part of the standard R installation. The most likely source for these packages is CRAN, but some are sourced from other places such as Bioconductor or various GitHub repositories.

Based on use patterns in multiple learning resources, several datasets contained in R add-on packages, such as the ones listed below, are also very popular.

6.4.1 penguins

Package: palmerpenguins

Description: Size measurements (bill length, bill depth, flipper length, body mass) for 344 penguins representing three species (Adélie, Chinstrap, and Gentoo) from three islands in the Palmer Archipelago, Antarctica.

Popularity: It was specifically designed as a modern, more ethically sound, and approachable replacement for the iris dataset. It is popular because it includes missing values (NAs) and more obvious species clusters, making it perfect for teaching data cleaning and visualization.

6.4.2 diamonds

Package: ggplot2

Description: This dataset contains the prices and other attributes (carat, cut, color, clarity, and dimensions) of nearly 54,000 round-cut diamonds.

Popularity: It is the primary dataset used to teach ggplot2. Because of its relatively large size (~54k rows), it is excellent for showing how to handle overplotting, identifying non-linear relationships, and demonstrating the impact of categorical variables on price.

6.4.3 flights

Package: nycflights13

Description: On-time data for all 336,776 flights that departed from New York City (JFK, LGA, EWR) in 2013. It is often used alongside other tables in the package like weather, planes, and airports.

Popularity: It is the “go-to” dataset for teaching data wrangling with dplyr. Its massive scale and relational structure (requiring joins between different tables) make it ideal for learning how to manipulate, filter, and summarize complex real-world data.

6.4.4 gapminder

Package: gapminder

Description: An excerpt of the Gapminder data on life expectancy, GDP per capita, and population by country and continent over time (from 1952 to 2007 in increments of 5 years).

Popularity: Made famous by Hans Rosling’s TED talks, this dataset is widely used to teach time-series visualization and animation (using gganimate). It helps users understand how to map aesthetics like size and color to multiple dimensions of data.

6.4.5 ISLR

Packages: ISLR and ISLR2

Description: Wage and other health/demographic data for 3,000 male workers in the Mid-Atlantic region of the US.

Popularity: As the companion dataset for the influential book An Introduction to Statistical Learning, it is the standard for teaching regression modeling, splines, and cross-validation. It provides a realistic scenario for predicting a continuous variable (income) using a mix of numerical and categorical predictors.

6.4.6 starwars

While the starwars dataset is technically an add-on dataset, it can also be referred to as a “special-interest” dataset.

Sometimes you’ll be drawn to a specific add-on dataset due to special personal or professional interests. For example, many people became fans of the Starwars franchise early in life so working with data about its characters is fun for them as adults.

Note

Information is given below for two special-interest datasets. Significant details are provided here as an example of how an LLM (in this case Gemini) can be used to provide a detailed description a dataset.

Package:

dplyr

Description:

The starwars dataset is a tibble (a modern version of a data frame) containing information on characters from the Star Wars films. It includes 14 variables:

Physical Traits: Name, height (cm), mass (kg), hair color, skin color, and eye color.

Demographics: Birth year, biological sex, gender, and homeworld.

Species: The species of the character (e.g., Human, Droid, Wookiee).

List-Columns: This is its most unique feature. It contains three columns—films, vehicles, and starships—where each “cell” actually contains a list of items rather than a single value.

Popularity:

The starwars dataset isn’t just popular because people like the movies; it is strategically designed to teach modern data science workflows that older datasets like iris simply cannot handle.

Teaching “List-Columns”

Most traditional datasets are “flat” (one value per cell). However, in the real world, data is often nested (like JSON). Because a character like Obi-Wan Kenobi appears in multiple films and flies multiple starships, those columns contain lists. This makes starwars the gold standard for teaching the nest(), unnest(), and purrr map functions.

Mastering Categorical Data

The dataset contains many categorical variables with many levels (like homeworld or species). It is excellent for practicing with the forcats package—learning how to lump rare categories together (e.g., “Other”) or reorder factors for better visualization.

Real-World “Messiness”

Unlike the perfectly curated iris dataset, starwars is intentionally “messy”:

Missing Values (NAs): Droids like R2-D2 don’t have “hair color” or a “birth year,” and Jabba the Hutt doesn’t have a “gender” in the same way humans do. This forces students to learn how to handle NA values using filter() or replace_na().

Variable Types: It includes character strings, numbers, and lists, providing a comprehensive workout for data cleaning.

The “Tidyverse” Native

Because it is a tibble, it behaves exactly as the Tidyverse expects. When you print it, it tells you the data types (e.g., <chr>, <int>, <list>) and only shows the first 10 rows, making it much more user-friendly for beginners than standard data frames that might flood the console with thousands of lines.

Note

Even though starwars is especially designed to illustrate features of Tidyverse packages, it’s also a good dataset for students to use if they want to first write code and base R and then rewrite the code using Tidyverse functions.

6.4.7 ebird_taxonomy

Another example of a special-interest dataset is the ebird_taxonomy dataset.

Package:

auk

Description:

This dataset provides a complete list of all bird species and recognizable forms (subspecies, hybrids, etc.) recognized by the Clements Checklist, which is the global taxonomic standard used by eBird.

It typically contains several key variables:

  • species_code: The unique, alphanumeric code for every bird (e.g., barswa for Barn Swallow). This is the “primary key” used to link datasets.

  • scientific_name: The Latin name (e.g., Hirundo rustica).

  • common_name: The English name used by birders.

  • Taxonomic Hierarchy: Columns for Order (e.g., Passeriformes) and Family (e.g., Hirundinidae), allowing users to group birds by biological relationship.

  • report_as: A critical column that helps “roll up” complex sightings (like subspecies) into a single species level for cleaner analysis.

Popularity:

The popularity of ebird_taxonomy stems from the fact that eBird is one of the largest biodiversity data sources in existence, with over a billion bird observations worldwide.

The “Dictionary” for Big Data

The raw eBird Basic Dataset (EBD) is massive (terabytes in size) and often uses codes instead of names to save space. Scientists and hobbyists use ebird_taxonomy as a look-up table to turn those cryptic codes into readable species names or to filter their data by specific families (e.g., “Show me all hawks and eagles”).

Solving the “Naming” Problem

Common names for birds change frequently, and different regions use different names for the same bird. This dataset provides the official, standardized mapping. It ensures that if one researcher calls a bird a “Common Eider” and another uses the Latin name, the auk package can reconcile them perfectly.

Filtering Specialized Observations

eBird data is “noisy.” It contains records of hybrids, “slashes” (e.g., “Slaty-backed/Vega Gull”), and domestic birds. The ebird_taxonomy dataset allows researchers to quickly filter for “species” only, removing the ambiguous entries that might skew statistical results in a population study.

Spatial and Temporal Integration

Because the taxonomy is updated annually, this dataset is the key to maintaining long-term consistency. When species are “split” (one species becomes two) or “lumped” by taxonomists, this dataset helps R users update their historical data to match modern scientific standards.

Note

The auk package is renowned for its use in ornithology. It has more than just the ebird_taxonomy datasets as well as important functions for performing data analysis.

While many (likely most) of the built-in datasets you’ll use will be “play” datasets for learning R, datasets in the auk package are examples of of how some datasets contained in R packages are used more for serious research.

6.5 Wrap-up

There’s a wide assortment of datasets available in many R packages. Typically, you’ll become aware of ones you’ll use most often through resources for learning R.

The datasets package contains some of the most often used and easily-accessed datasets for learning R. The vast number of R add-on packages offer both popular high-used as well as special interest datasets.

Regardless of its source, it’s a good practice to gather at least a minimal amount of information about the dataset before using it.

You’ll often discover datasets through tutorials, course materials, or recommendations from others. If those sources aren’t enough, you may also search for datasets online or use your favorite large language model to suggest options.
